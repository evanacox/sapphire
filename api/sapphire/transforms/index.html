Module sapphire::transforms

source ·
Expand description

Defines the “transform” passes in SIR’s infrastructure.

These are the passes that can (potentially) modify SIR, and don’t actually logically yield a result.

Some of these “transforms” are not actually transformations (e.g. the verify pass is a “transform pass” even though it manipulates no IR), but most of them are. All of them logically yield no result except the IR that exists after they run.

  • gvn
    Runs a global value-numbering algorithm over func to remove redundant expressions.
  • mem2reg
    Promotes function-local memory (that doesn’t escape) into φ nodes where possible.

    Performs critical edge splitting over a function. Each edge that is split has a dummy node inserted that contains only an unconditional branch to the block that was originally being targeted.
